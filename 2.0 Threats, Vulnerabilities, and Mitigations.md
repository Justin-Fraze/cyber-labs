# 2.0 Threats, Vulnerabilities, and Mitigations

## Cross-site scripting
XSS - originally came from browser security flaws. Info from one site can be shared with another.
One of the most common web app vulnerabilities. 
<img width="1024" height="559" alt="image" src="https://github.com/user-attachments/assets/8567ef1a-d964-4e08-98b5-96ed25a6afe4" />

There are different types of XSS attacks. 
Non-persistent (reflected) XSS attack:
The website allows scripts to run in user input, commonly a search box. Often, an attacker will use an email scam to send a link that takes advantage of this vulnerability.
Attacker then uses credentials/session IDs/cookies to steal victim information.

Reflective type and persistent (stored) type.
Persistent type example might be an attacker posting a message on social media with a malicious payload.

Protecting yourself aganist a XSS:
* Be careful whne clicking untrusted links.
* Consider disabling JavaScript
* Validate input fields.

## Hardware vulnerabilities

The many devices in our home and workplace are all potential security issues. IoT is everywhere.

* Firmware refers to the software inside of the hardware. It is the operating system of the operating device. This leads to vendors being the only the only ones who can patch or fix the hardware, and they are rarely security focused.
* End of Life (EOL) - refers to when the manufacturer stops selling the product. 
* End of service life (EOSL) - when the manufacturer stops giving support, security patches, or updates for the device. Thus, EOSL tech is a serious concern if it is in your office.
* Legacy platforms - old devices and operating systems that may be running risky, non-patched software.
